# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

`@mrmartineau/otter-web` is the main package of the Otter monorepo: a self-hosted bookmark manager + media tracker. The React SPA frontend **and** the Hono API run in a single Cloudflare Worker. Data lives in Postgres (accessed via Drizzle ORM through a Hyperdrive binding). Auth is `better-auth`. The repo root is two levels up (`../../`).

## Commands

Run from this package (`packages/web`) unless noted.

```sh
pnpm dev              # vite dev server (wrapped in `portless run`)
pnpm build            # tsc -b && vite build  (also the deploy build step)
pnpm deploy           # build + wrangler deploy
pnpm preview          # build + vite preview

pnpm db:generate      # drizzle-kit generate — create migration from db/schema.ts
pnpm db:migrate       # drizzle-kit migrate — apply migrations
pnpm db:studio        # drizzle-kit studio
pnpm cf-typegen       # wrangler types — regenerate worker-configuration.d.ts after binding changes

# From repo root (../../):
pnpm check            # biome check --write (lint + format, whole monorepo)
pnpm web:dev          # = packages/web dev
pnpm web:build        # = packages/web build
```

**Type-checking:** use `tsc -b` (the project-references build). Plain `tsc --noEmit` / the `type-check` script checks nothing here — the real config lives in `tsconfig.app.json` / `tsconfig.node.json`.

**Tests:** vitest, environment `happy-dom`. No test script in package.json — invoke directly:
```sh
pnpm vitest run                       # all tests once
pnpm vitest run worker/scraper        # a dir
pnpm vitest run src/utils/filteredTags.test.ts   # one file
pnpm vitest                           # watch mode
```
Tests are colocated as `*.test.ts` next to source.

**Package manager:** pnpm. corepack may error on version mismatch — pass `--pm-on-fail=ignore` if so.

## Architecture

### Single Worker, two concerns
`worker/index.ts` exports the default fetch handler → `worker/hono.ts`. Two Hono apps there:
- `api` — mounted at `/api`, all backend routes.
- `app` — top-level: handles `/.well-known/*` OIDC discovery, mounts `api`, and `notFound` serves static assets (the built SPA) falling back to `index.html` for client-side routing. `wrangler.jsonc` sets `run_worker_first: true` so the Worker sees every request before assets.

Routes that don't need the DB (`/`, `/scrape`, `/ai/*`, `/rss`) are declared **before** `api.use('*', dbMiddleware)` so they skip the per-request Postgres connection. Add new DB-free routes above that line.

### Database
- Schema: `db/schema.ts` (Drizzle, ~25 pgTables: auth tables, `oauth*`, `profiles`, `bookmarks`, `tags`/`bookmarkTags`, `media`, `toots`, `tweets`, `feeds`, `shares`, `userIntegrations`). Edit schema → `db:generate` → `db:migrate`.
- `db/client.ts` builds a `pg` Client + drizzle instance. Connection string comes from the `HYPERDRIVE` binding (prod) or `DATABASE_URL` (local), via `getDatabaseUrl`.
- `worker/middleware/db.ts` opens a fresh connection per request, sets `c.var.db`, and closes it in `waitUntil` after the response. Handlers read the DB off context, never construct their own client.

### Auth & request context
- `auth/server.ts` — `createAuth(env, db)` configures `better-auth` (drizzle adapter, JWT plugin, `@better-auth/oauth-provider`). Sign-up is disabled unless `BETTER_AUTH_DISABLE_SIGNUP === 'false'`.
- `worker/context.ts` — `requireRequestContext(c)` / `getRequestContext` resolve the caller's profile via **three** auth methods: session cookie, `Bearer` OAuth access token (JWT verified against the JWKS), or an `api_key` UUID. Returns `{ db, profile, user }`. Backend handlers start by calling `requireRequestContext`.
- OAuth provider exists to serve the Raycast extension and MCP clients (`worker/mcp/`).

### Frontend
- Vite + React 19 SPA. `src/main.tsx` → `src/router.tsx` (TanStack Router).
- **File-based routing** in `src/routes/`. `routeTree.gen.ts` is auto-generated by `@tanstack/router-plugin` (Vite) — never edit it; it's git-ignored from biome. `_app/` = authed routes, `_public/` = unauthed.
- Server state via TanStack Query (persisted to async-storage). URL state via `nuqs`.
- API client functions live in `src/utils/fetching/*` (one file per resource). Paginated endpoints use `apiResponseGenerator` (`src/utils/fetching/apiResponse.ts`) producing `{ data, count, _links }` HATEOAS-style envelopes.
- Components in `src/components/`, each often with a colocated `.css`. Styling: Tailwind v4 (via `@tailwindcss/vite`) + PostCSS. Radix UI primitives. The shared `@mrmartineau/kit` design kit (catalog dep) provides the biome config too.
- Path alias `@/` → `src/`.

### Content extraction & integrations
`worker/scraper/` and `worker/xtractr/` scrape/extract page content (defuddle + linkedom). `worker/ai/` calls Cloudflare Workers AI (`AI` binding) for title/description/summarise/classify. `worker/toots/`, `worker/bluesky/`, `worker/rss/`, `worker/social.ts` handle Mastodon/Bluesky/RSS/Twitter import + posting.

## Conventions
- **Lint/format:** Biome (config extends `@mrmartineau/kit`). Run `pnpm check` from root. Single quotes, no semicolons.
- **Bindings:** after editing `wrangler.jsonc` bindings, run `cf-typegen` to refresh types.
- **Commits:** Conventional Commits — drive semantic-release per-package versioning. `fix:` → patch, `feat:` → minor, `feat!:`/`BREAKING CHANGE:` → major. Scope web changes with `(web)`.
- This is a pnpm workspace; sibling packages (`app`, `web-extension`, `raycast-extension`, `chrome-extension`) consume this Worker's API.
